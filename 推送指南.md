## 💻 第二步：在本地终端执行命令

### Windows 用户

1. **打开 PowerShell 或 Git Bash**
   - 按 `Win + X`，选择 "Windows PowerShell"
   - 或搜索 "Git Bash"（如已安装 Git）

2. **进入项目目录**
   ```bash
   # 替换为您解压的路径
   cd C:\Users\您的用户名\Downloads\photography-website
   ```

3. **初始化 Git 仓库**
   ```bash
   git init
   git add .
   git commit -m "Initial commit: 摄影作品发布网站"
   git branch -M main
   ```

### Mac/Linux 用户

1. **打开终端**
   - Mac: 按 `Cmd + 空格`，搜索 "Terminal"
   - Linux: 按 `Ctrl + Alt + T`

2. **进入项目目录**
   ```bash
   # 替换为您解压的路径
   cd ~/Downloads/photography-website
   ```

3. **初始化 Git 仓库**
   ```bash
   git init
   git add .
   git commit -m "Initial commit: 摄影作品发布网站"
   git branch -M main
   ```

## 🚀 第三步：创建 GitHub 仓库

1. 访问 https://github.com/new
2. 填写信息：
   - **Repository name**: `photography-website`
   - **Description**: 光影之间 - 摄影作品发布网站
   - **Public**（公开）
   - **不要勾选** "Add a README file"
3. 点击 **Create repository**

---

## 📤 第四步：推送代码到 GitHub

**在本地终端执行**（替换 `您的用户名`）：

```bash
# 添加远程仓库
git remote add origin https://github.com/您的用户名/photography-website.git

# 推送代码
git push -u origin main
```

如果提示输入用户名和密码：
- **Username**: 输入您的 GitHub 用户名
- **Password**: 输入 Personal Access Token（不是密码）
  - 创建 Token: https://github.com/settings/tokens/new
  - 勾选 `repo` 权限

---

## 🌐 第五步：启用 GitHub Pages

1. 进入 GitHub 仓库页面
2. 点击 **Settings** 标签
3. 左侧菜单找到 **Pages**
4. 配置：
   - **Source**: Deploy from a branch
   - **Branch**: main
   - **Folder**: / (root)
5. 点击 **Save**
6. 等待 1-2 分钟

---

## 🔧 常见问题

### Q1: 推送时提示 "Authentication failed"

**解决方案**：使用 Personal Access Token

1. 创建 Token: https://github.com/settings/tokens/new
2. 勾选 `repo` 权限，生成并复制 Token
3. 推送时密码处粘贴 Token

或使用 Token 方式推送：
```bash
# 替换变量
TOKEN="您的Token"
USERNAME="您的用户名"

git remote set-url origin https://oauth2:${TOKEN}@github.com/${USERNAME}/photography-website.git
git push -u origin main
```

### Q2: 提示 "git 不是内部或外部命令"

**解决方案**：安装 Git

- Windows: https://git-scm.com/download/win
- Mac: `brew install git`
- Linux: `sudo apt install git`

### Q3: Pages 显示 404

**解决方案**：
1. 检查分支是否为 `main`
2. 检查根目录是否有 `index.html`
3. 等待 2-3 分钟后刷新
